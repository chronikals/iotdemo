The black ABB bar

The black ABB bar provides consistency and familiarity across ABB’s software interfaces, and typically contains logo, product name, main navigation and critical information. In addition to being an important branding element, it is helpful for the user’s perception of system structure and navigation.

Menus

Tabs are useful for allowing users to easily switch between different views and to clearly see what the browsing choices are. They can be used as top-level navigation, or to browse categorized data sets within a panel. Care should be taken to not provide too many nested levels of tabs, or to use tabs when there are too many alternatives.

Selectors

Checkboxes

Use when you want the user to select 0 or more options from a predefined list. The class input[type="checkbox"] is used here.

Radio buttons

Use when you want the user to select just 1 option from a list. The class input[type="radio"] is used here.

Buttons

There are 3 types of buttons: Normal [btn-default in Bootstrap], Commit [btn-primary in Bootstrap] and Discrete [btn-link in Bootstrap]. In order to improve contrast and readability, additional classes are created for alternative background colors used in the user interface. This stylesheet specifies the styling for buttons used on gray, blue, dark or black backgrounds. Suffixes are added to the base classes when defining the buttons in the HTML code.

Buttons light background

For light or white background the original classes from Boortrap are used. For a Nomal button use the btn-default, for a Commit button use btn-primary and for a Discrete button use btn-link

Dropdowns

Drop-down lists are used to make one selection from a limited number of items. Since the other choices are hidden, it should be used when the drop-down content is predictable for the user. In a similar way from the buttons, dropdowns and combo boxes (select) use the default Bootstrap classes for light backgrounds and are added preffixes for blue and black backgrounds.

Dropdown light background

The dropdowns inherit the styling from the button classes and add the class dropdown-toggle. For example a Normal dropdown like the one in the examples below is styled using the btn-default dropdown-toggle class. For the combo boxes (select) the styling is inherited from the form-control classes.

Notifications

Notifications are placed on top of the content, and may be displayed in two different ways: as a notification banner, and as a confirmation box

Notification banner

Notification banners may be removed by user, or by time-out. These banners use Bootstrap default, the alert classes together with an indication of the type of notification, which can be information [alert-info], success [alert-success], warning [alert-warning] or error [alert-error].

× Success! Indicates a successful or positive action.
× Info! Indicates a neutral informative change or action.
× Warning! Indicates a warning that might need attention.
× Error! Indicates a dangerous or potentially negative action.

Confirmation boxes

Confirmation boxes must be acknowledged by the user. Since these are not standard elements defined in Bootstrap, it is up to the development teams to decide which library to use and perform the styling according to CommonUX recommended guidelines. As an example, we use here the Pnotify bootstrap library to show how confirmation boxes can look like.
